# 2026-07-02 工作流升级优化整改方案

## 1. 背景

用户反馈新系统在项目新建、列表顺序、专家评分入口和交付说明上仍不够直观。结合旧系统导出的项目评审记录、项目统计列表，以及线上新系统回归结果，本轮目标是让新系统不只是“能用”，而是更适合交付培训和实际办理。

## 2. 已执行整改

| 优先级 | 模块 | 整改内容 | 状态 |
| --- | --- | --- | --- |
| P0 | 项目列表 | 增加序号，默认按创建时间从早到晚 | 完成 |
| P0 | 项目申报 | 预算改为“万元”录入，显示系统保存金额（元） | 完成 |
| P0 | 审核任务 | 操作按钮改为“详情 / 审核处理”文字按钮 | 完成 |

| P0 | 终审 | 管理员终审记录是否推荐、是否支持、支持方式、支持金额、推荐专家 | 完成 |

| P1 | 文档 | 更新截图版操作手册、测试报告、部署文档 | 完成 |

## 3. 旧系统能力吸收

| 旧系统能力 | 新系统处理 |
| --- | --- |
| 专家评分按多个维度打分 | 使用数据字典 `expert_review_criterion` 配置评分项、分组、满分、说明 |
| 项目统计表包含支持金额、推荐专家等字段 | 项目导出 CSV 增加是否推荐、是否支持、支持方式、支持资金（万元）、推荐专家 |
| 项目办理顺序按时间理解 | 列表增加序号，项目/账号/消息默认按时间正序 |
| 申报表字段较多但流程明确 | 新系统使用工作台抽屉，分为基本信息、项目概述、负责人/成员、经费预算、设备材料、盖章承诺、附件材料 |

## 5. 仍建议后续增强

这些不阻塞本轮上线，但建议作为下一轮 P1/P2：

| 优先级 | 建议 | 说明 |
| --- | --- | --- |
| P1 | 申报批次模板化 | 不同批次可配置不同申报字段、附件清单和预算科目 |
| P1 | 专家分配机制 | 当前按角色阶段流转，后续可细化为指定专家组/专家池 |
| P1 | 评分汇总报表 | 管理员可按项目查看专家均分、分项得分、意见汇总 |
| P2 | 正式 CI 截图归档 | 每次部署后自动产出 HTML 报告和截图附件 |
| P2 | 测试数据归档治理 | `E2E-` 数据按批次归档，不进入默认业务统计 |
| P2 | 权限矩阵细化 | 按按钮/接口细化权限，例如导出、评分配置、终审支持字段 |

## 6. 上线建议

1. 合并并推送本轮代码。
2. 按部署文档执行宝塔部署。
3. 部署后强刷浏览器。
4. 用单位、区县、部门、专家、普通管理员、超级管理员依次登录。
5. 重点验收：
   - 项目列表序号和时间正序。
   - 新建项目预算金额（万元）。
   - 专家“审核处理”评分弹窗。
   - 管理员终审支持信息。
   - 项目导出 CSV 字段。
6. 保留全部 `E2E-` 测试数据。

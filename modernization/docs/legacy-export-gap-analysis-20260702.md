# 旧系统导出表差异分析与整改方案

生成时间：2026-07-02

参考文件：

- `项目评审记录2026-06-25.xls`
- `项目统计列表2026-06-25.xls`

## 1. 已识别差异

### 1.1 项目评审记录

旧系统“项目评审记录”导出包含：

- 基础字段：编号、项目名称、计划类别、申报单位、专家、评分时间、总分。
- 专家评分维度：按大类拆分为政策符合性评价、项目成果及技术水平评价、产学研合作评价、项目实施条件评价、项目预期效益评价、经费预算。
- 真正得分子项共 17 项，总分 100 分。

新系统原状态：

- 只有 `project_reviews.score` 一个总分字段和 `comment` 审核意见。
- 专家评分弹窗只有单个评分输入框。
- 导出结果只导出总分，不能追溯每个维度。

本次整改：

- 新增专家评分项配置，使用数据字典分组 `expert_review_criterion`。
- 评分项支持：评分大类、满分、说明、启停、排序。
- 专家审核弹窗按大类展示逐项评分，系统自动汇总总分。
- 后端按配置重新校验并计算总分，防止前端篡改总分。
- 评审结果导出在“评分”后展开每个评分维度列。
- 历史评审数据继续可查；没有逐项评分的历史记录，维度列为空。

### 1.2 项目统计列表

旧系统“项目统计列表”导出包含：

- 编号、项目名称、计划类别、申报单位。
- 归口管理单位、归属区域。
- 提交时间、状态。
- 是否推荐、是否支持、支持资金（万元）、推荐专家。

新系统当前覆盖情况：

| 旧字段 | 新系统状态 | 说明 |
| --- | --- | --- |
| 编号 | 已有 | `projects.id`，旧数据可保留 `legacy_id` |
| 项目名称 | 已有 | `projects.title` |
| 计划类别 | 部分覆盖 | 当前用申报批次、项目类别/类型承接 |
| 申报单位 | 已有 | `projects.unit_id` |
| 归口管理单位 | 已补充导出 | 项目 metadata 中的 `management_unit` 会进入项目导出 |
| 归属区域 | 已补充导出 | 优先使用项目 metadata 中的 `region_code`，否则使用单位 `region_code` |
| 提交时间 | 已有 | `submitted_at` |
| 状态 | 已有 | `status` |
| 是否推荐 | 已补充 | 管理员终审支持信息中记录 |
| 是否支持 | 已补充 | 管理员终审支持信息中记录 |
| 支持资金（万元） | 已补充 | 管理员终审支持信息中记录并导出 |
| 推荐专家 | 已补充 | 可由终审录入，留空时汇总专家推荐记录 |

## 2. 本次已完成整改

1. 项目类别/项目类型来源已说明并优化：
   - 全局候选项在“数据字典”维护：`project_category`、`project_type`。
   - 批次允许范围在“申报批次”维护：允许项目类别、允许项目类型。
   - 项目表单优先显示当前批次允许范围。

2. 预算金额口径已明确：
   - 前台项目申报表单按“万元”录入，降低业务填报成本。
   - 后端和历史兼容导出仍保留预算金额（元）。

3. 保存草稿无反馈已修复：
   - 后端 422 错误会显示中文提示。
   - 类别/类型不在当前批次允许范围内时，前端不再静默失败。

4. 列表顺序已调整：
   - 账号管理、站内消息默认按创建时间从早到晚。
   - 列表增加序号，便于交付演示和业务核对。

5. 专家评分维度已补齐：
   - 超管/管理员可在“数据字典”维护评分项。
   - 专家在“审核任务”中逐项评分。
   - 后端强校验并计算总分。
   - 审核结果导出包含维度列。

6. 项目统计导出字段已补齐：
   - 增加计划类别、归口管理单位、归属区域。
   - 增加是否推荐、是否支持、支持方式、支持资金（万元）、推荐专家。
   - 导出默认按项目创建时间从早到晚，便于业务核对。

7. 系统配置密钥写入修正：
   - 单项配置更新统一使用 `Crypt::encryptString()`。
   - 运行时读取、掩码显示和空密码保留逻辑一致。

## 3. 建议下一轮整改

### P1 专家分配机制

- 当前专家按角色查看专家阶段任务。
- 建议增加“指定专家/专家组”：
  - 管理员或部门在流转到专家前选择专家。
  - 专家只看到分配给自己的项目。
  - 项目统计表可直接输出推荐专家。

### P2 归口管理和区域快照

- 项目创建时固化申报单位当时的区域、归口管理单位。
- 避免单位资料后续修改影响历史统计。
- 项目统计导出优先使用项目快照字段。

### P2 旧系统导出对齐

- 增加“旧系统格式导出”模板：
  - 项目评审记录导出。
  - 项目统计列表导出。
- 表头名称、单位、排序与旧系统保持一致，降低业务人员切换成本。

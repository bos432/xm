## 2. 标准部署命令

在宝塔终端执行：

```bash
cd /www/wwwroot/nxm.zlck888.com

APP_ROOT=/www/wwwroot/nxm.zlck888.com \
REPO_URL=https://github.com/bos432/xm.git \
BRANCH=main \
PHP_BIN=/www/server/php/83/bin/php \
FRONTEND_API_BASE=/api \
RUN_SEED=1 \
bash ./deploy.sh
```

正常日志最后应看到：

```text
Deployment complete: <release-id>
Website root should be: /www/wwwroot/nxm.zlck888.com/current/public
```

说明：

- `FRONTEND_API_BASE=/api` 表示前端用当前域名下的 `/api` 访问后端。
- 不要设置成旧域名，例如 `https://rd.chaimen666.com/api`，否则验证码和登录会跨域失败。

## 3. 部署后健康检查

```bash
cd /www/wwwroot/nxm.zlck888.com/current/backend

/www/server/php/83/bin/php artisan health:login-check \
  --username=health_check_user \
  --password='HealthCheck-2026'

curl -s https://nxm.zlck888.com/api/auth/captcha; echo
curl -s https://nxm.zlck888.com/api/public/homepage | head -c 500; echo
curl -I https://nxm.zlck888.com/
```

通过标准：

- `Health login passed for [health_check_user].`
- 验证码接口返回 `captcha_id` 和算术题。
- 首页内容接口返回 JSON。
- 首页返回 `HTTP/2 200`。

## 4. 健康检查账号配置

如果部署日志出现：

```text
Health login credentials missing; local login check skipped.
```

执行：

```bash
cd /www/wwwroot/nxm.zlck888.com

cp shared/.env shared/.env.bak-$(date +%Y%m%d%H%M%S)

grep -q '^APP_URL=' shared/.env \
  && sed -i 's#^APP_URL=.*#APP_URL=https://nxm.zlck888.com#' shared/.env \
  || echo 'APP_URL=https://nxm.zlck888.com' >> shared/.env

grep -q '^HEALTH_CHECK_USERNAME=' shared/.env \
  && sed -i 's#^HEALTH_CHECK_USERNAME=.*#HEALTH_CHECK_USERNAME=health_check_user#' shared/.env \
  || echo 'HEALTH_CHECK_USERNAME=health_check_user' >> shared/.env

grep -q '^HEALTH_CHECK_PASSWORD=' shared/.env \
  && sed -i 's#^HEALTH_CHECK_PASSWORD=.*#HEALTH_CHECK_PASSWORD=HealthCheck-2026#' shared/.env \
  || echo 'HEALTH_CHECK_PASSWORD=HealthCheck-2026' >> shared/.env

cd /www/wwwroot/nxm.zlck888.com/current/backend
/www/server/php/83/bin/php artisan config:clear
/www/server/php/83/bin/php artisan config:cache
```

## 5. 队列常驻运行

队列用于异步任务，例如通知、邮件、后台任务。建议放入宝塔 Supervisor 常驻。

推荐命令：

```bash
cd /www/wwwroot/nxm.zlck888.com/current/backend && /www/server/php/83/bin/php artisan queue:work database --queue=default --tries=3 --timeout=90 --sleep=3
```

宝塔 Supervisor 建议配置：

| 项目 | 内容 |
| --- | --- |
| 名称 | `nxm-queue-default` |
| 运行目录 | `/www/wwwroot/nxm.zlck888.com/current/backend` |
| 启动命令 | `/www/server/php/83/bin/php artisan queue:work database --queue=default --tries=3 --timeout=90 --sleep=3` |
| 进程数量 | `1` |
| 自动启动 | 开启 |
| 自动重启 | 开启 |

检查命令：

```bash
ps -ef | grep 'queue:work' | grep -v grep
```

## 8. 回滚

查看历史发布目录：

```bash
cd /www/wwwroot/nxm.zlck888.com
ls -lt releases
```

回滚到上一个发布目录：

```bash
cd /www/wwwroot/nxm.zlck888.com
ln -sfn releases/<previous-release-id> current
nginx -t && /etc/init.d/nginx reload
```

注意：

- 本轮没有新增数据库迁移，常规代码回滚风险较低。
- 如果执行过“归档测试批次”，代码回滚不会自动恢复批次状态，需要后台或数据库手动处理。

## 9. 部署异常说明

- Composer 阿里云镜像 `404` 后继续下载成功：不影响部署。
- `PHP Warning: Module "mbstring" is already loaded`：PHP 配置重复加载扩展，当前不影响运行。
- Vite `INVALID_ANNOTATION`：第三方库构建提示，不影响前端产物。
- 大 chunk warning：性能治理项，不阻塞上线。

# 常见问题与故障处理

## 1. GitHub 拉取失败

现象：

```text
Invalid username or token. Password authentication is not supported.
```

原因：

- 仓库私有时需要 Token。
- 之前终端粘贴 Token 容易把命令一起粘进去。

当前处理：

- 仓库已公开。
- 部署直接使用：

```bash
REPO_URL=https://github.com/bos432/xm.git
```

不再需要 GitHub Token。

## 2. 首页接口 500

先检查：

```bash
curl -s https://nxm.zlck888.com/api/auth/captcha
curl -s https://nxm.zlck888.com/api/public/homepage | head -c 500
```

查看日志：

```bash
cd /www/wwwroot/nxm.zlck888.com/current/backend
tail -n 200 storage/logs/laravel*.log
```

常用修复：

```bash
cd /www/wwwroot/nxm.zlck888.com/current/backend
/www/server/php/83/bin/php artisan config:clear
/www/server/php/83/bin/php artisan route:clear
/www/server/php/83/bin/php artisan cache:clear
/www/server/php/83/bin/php artisan config:cache
/www/server/php/83/bin/php artisan route:cache
```

## 3. 登录被锁定或限流

现象：

```text
登录失败次数过多，请稍后再试或联系管理员解锁
```

处理：

1. 用超级管理员登录。
2. 进入“安全中心”。
3. 按账号或 IP 搜索。
4. 解锁账号或 IP。
5. 验收期间可临时加入测试白名单 IP。

注意：

- 不建议关闭登录风控。
- 只允许临时白名单或临时放宽。

## 4. 验证码怎么填

验证码是算术题：

```text
8 + 1 = ?
```

填写：

```text
9
```

## 5. Favicon 不显示

检查：

```bash
curl -s https://nxm.zlck888.com/api/public/homepage | grep favicon
```

如果返回：

```json
"favicon_url":null
```

说明后台还没有上传站点图标。

处理：

1. 超管登录。
2. 进入 `首页管理 -> 品牌素材`。
3. 上传 Favicon。
4. 刷新或重新打开浏览器标签页。

## 6. 队列不运行

检查：

```bash
ps -ef | grep 'queue:work' | grep -v grep
```

临时启动：

```bash
cd /www/wwwroot/nxm.zlck888.com/current/backend
/www/server/php/83/bin/php artisan queue:work database --queue=default --tries=3 --timeout=90 --sleep=3
```

长期运行：

- 放到宝塔 Supervisor。
- 开启自动启动和自动重启。

## 7. Composer 阿里云镜像 404

部署日志可能出现：

```text
Failed downloading ..., trying the next URL
```

只要后面继续 `Installing ... Extracting archive` 并最终部署完成，就不是失败。

## 8. PHP mbstring already loaded

现象：

```text
PHP Warning: Module "mbstring" is already loaded
```

说明 PHP 配置里重复加载了 mbstring。当前不影响运行，可以后续在 PHP 配置中清理重复扩展。


## 10. 回滚后页面还是旧缓存

处理：

1. 强刷浏览器。
2. 清理前端缓存。
3. 检查首页 `last-modified`：

```bash
curl -I https://nxm.zlck888.com/
```

4. 确认 Nginx 网站根目录指向：

```text
/www/wwwroot/nxm.zlck888.com/current/public
```

## 11. 验证码一直加载中

现象：

- 首页登录框显示验证码加载中。
- 服务器直接访问 `/api/auth/captcha` 正常。
- 浏览器控制台出现跨域错误，类似：

```text
Access to fetch at 'https://rd.chaimen666.com/api/auth/captcha'
from origin 'https://nxm.zlck888.com' has been blocked by CORS policy
```

原因：

- 前端构建时读取到了旧环境变量 `VITE_API_BASE=https://rd.chaimen666.com/api`。
- 构建产物把接口地址写死到旧域名，导致当前站点跨域请求失败。

修复：

```bash
cd /www/wwwroot/nxm.zlck888.com

unset VITE_API_BASE

APP_ROOT=/www/wwwroot/nxm.zlck888.com \
REPO_URL=https://github.com/bos432/xm.git \
BRANCH=main \
PHP_BIN=/www/server/php/83/bin/php \
VITE_API_BASE=/api \
FRONTEND_API_BASE=/api \
RUN_SEED=1 \
bash ./deploy.sh
```

部署完成后检查：

```bash
grep -R "rd.chaimen666.com" -n /www/wwwroot/nxm.zlck888.com/current/public/assets || echo "OK: no old API domain"
curl -s https://nxm.zlck888.com/api/auth/captcha; echo
```

长期处理：

- 部署脚本已固定默认 `FRONTEND_API_BASE=/api`。
- 后续部署完成后可把新脚本复制到站点根目录：

```bash
cd /www/wwwroot/nxm.zlck888.com
cp current/source/modernization/deploy/deploy.sh ./deploy.sh
chmod +x ./deploy.sh
```

# 科技项目管理系统交付验收测试方案（逐步版）

## 1. 测试目标

本测试方案用于交付前人工验收，覆盖首页、登录、单位申报、区县审核、部门审核、专家评分、业务管理员、超级管理员、部署健康检查和安全配置。

测试站点：

https://nxm.zlck888.com/

测试原则：

- 使用测试账号和 `E2E-` 测试数据。
- 不物理删除测试数据。
- 不随意提交专家评分或审批动作，除非专门创建新的测试项目。
- 发现问题先记录页面、账号、操作步骤和截图。

## 3. 测试前服务器健康检查

在宝塔终端执行：

```bash
cd /www/wwwroot/nxm.zlck888.com/current/backend

/www/server/php/83/bin/php artisan health:login-check \
  --username=health_check_user \
  --password='HealthCheck-2026'

curl -I https://nxm.zlck888.com/
curl -I https://nxm.zlck888.com/dashboard
curl -I https://nxm.zlck888.com/application-batches
curl -I https://nxm.zlck888.com/public-home
```

预期结果：

- `Health login passed for [health_check_user].`
- 所有 `curl -I` 返回 `HTTP/2 200`。

## 12. 验收流程测试（可选）

前提：项目已通过或处于允许验收状态。

### 12.1 单位提交验收

1. 单位账号登录。
2. 进入验收管理。
3. 点击发起验收。
4. 选择项目。
5. 上传材料。

建议检查材料分类：

- 验收申请书
- 项目总结
- 财务材料
- 成果证明
- 其他

6. 故意缺少必传材料提交一次。

预期结果：

- 前端提示缺失材料。
- 后端返回明确中文提示。

7. 补齐材料后提交。

### 12.2 分角色处理验收

1. 区县账号处理验收初审。
2. 部门账号处理验收复核。
3. 专家账号处理验收评审。
4. 管理员或超管处理终审关闭。

## 13. 最终验收通过标准

满足以下条件即通过：

- 首页正常访问。
- 验证码正常加载。
- 六类角色均可登录。
- 非超管看不到系统开发、迁移、配置类信息。
- 超管可见系统配置、安全中心、迁移准备。
- 申报批次页面不白屏且有数据。
- 项目 URL 筛选参数生效。
- 验收 URL 筛选参数生效。
- 专家评分入口存在，多维评分正常。
- 富文本编辑器可用。
- 安全中心可查看风控状态。
- 部署健康检查通过。
- 测试数据保留，不物理删除。
